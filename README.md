# XTon 数据格式文档

## \[中\|[English](README-E.md)]

## 简介

`XTon` 是一种轻量级的数据格式，旨在通过简化语法，提供比 JSON 更简洁的方式来表示和传输数据。全称`Xiao Tian Object Notation`（**小田对象符号表示法**），与 JSON 类似，`XTon` 支持多种数据类型，如字符串、数字、布尔值、数组和对象。它的设计目标是简化数据表达，减少冗余字符，并提供更易读、易用的数据表示方式。

`XTon` 格式的特点是通过使用简单的符号和转义规则来表达数据结构，同时避免了 JSON 中常见的花括号、引号、逗号等符号的使用。

## 数据类型

`XTon` 格式支持以下几种数据类型：

- **字符串**：通过反引号 `'` 来表示字符串。

- **数字**：直接表示数字，不需要引号。

- **布尔值**：布尔值使用 `\true` 或 `\false` 来表示。

- **空值**：空值使用 `\none` 来表示。

- **数组**：通过中括号 `[]` 来表示数组，支持嵌套。

- **对象**：通过 `< >` 来表示对象，使用 `/` 来分隔键和值。

## 转义规则

在 `XTon` 格式中，以下字符需要进行转义：

- `-`：通过 `\-` 来转义。

- `<`：通过 `\<` 来转义。

- `>`：通过 `\>` 来转义。

- `[` ：通过 `\[` 来转义。

- `]`：通过 `\]` 来转义。

- `/`：通过 `\/` 来转义。

这些字符在 `XTon` 格式中作为语法符号使用，转义后可以确保它们在字符串中被正确解析。

## 示例

以下是 `XTon` 格式的一些基本示例，演示了如何表示不同类型的数据。

### 字符串

- 原始字符串：`'23.5`

- 解释：`"23.5"`（字符串）

### 数字

- 数字：`23.5`

- 解释：`23.5`（数字）

### 布尔值

- 布尔值：`\true`

- 解释：`true`（布尔值）

### 空值

- 空值：`\none`

- 解释：`null`（空值）

### 数组

- 数组：`[a/25.3/87]`

- 解释：`["a", 25.3, 87]`（数组）

### 对象

- 对象：

```xton
 q-<a-\none/k-\true/g-[a/25.3/87]>
```

- 解释：

```json
{
    "q": {
        "a": null,
        "k": true,
        "g":  ["a",  25.3,  87]
    }
}
```

## 使用场景

`XTon` 格式适用于对数据大小和传输效率要求较高的场景，尤其是在嵌入式系统、设备通信、轻量级数据存储等方面。

- **嵌入式系统**：对于内存和存储受限的设备，`XTon` 提供了比 JSON 更简洁的表示方式。

- **设备通信**：在设备间传输数据时，`XTon` 格式可以减少数据包的大小，提高传输效率。

- **数据存储**：对于一些需要简化数据存储格式的应用，`XTon` 提供了更紧凑的表示方式。

## 语法总结

1. **字符串**：使用反引号 `'` 表示，例如 `'text`。

2. **数字**：直接表示数字，例如 `23.5`。

3. **布尔值**：使用 `\true` 或 `\false` 来表示布尔值。

4. **空值**：使用 `\none` 来表示空值。

5. **数组**：使用中括号 `[]` 表示数组，数组元素通过 `/` 分隔，例如 `[a/25.3/87]`。

6. **对象**：使用尖括号 `< >` 表示对象，键值对通过 `/` 分隔，例如 `q-<a-\none/k-\true/g-[a/25.3/87]>`。

## 转义字符

在 `XTon` 格式中，以下字符需要进行转义：

- `-` → `\-`

- `<` → `\<`

- `>` → `\>`

- `[` → `\[`

- `]` → `\]`

- `/` → `\/`

- `\` → `\\`

- `'` → `\'`
